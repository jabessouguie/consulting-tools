## Commentaire Long

Ce qui m'a le plus marqué dans nos missions récentes, c'est que l'IA a déplacé le curseur de valeur. Avant, un bon data scientist = quelqu'un qui code vite et propre. Maintenant, c'est quelqu'un qui pose les bonnes questions et sait challenger les outputs. On a gagné 40% de temps sur le code chez un client retail, mais on passe ce temps à creuser les insights métier et à itérer sur les hypothèses. Le profil "data scientist consultant" prend tout son sens : la technique devient un outil, pas la finalité.
